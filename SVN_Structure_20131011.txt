SVN 사용 관련 회의 - 2013. 10. 11. AM 9:30 / 연구소장님실


   0. SVN 폴더 구조는 다음과 같이 사용한다.
       (최상위 폴더는 archive, practice, quintet 으로 구성된다.)

http://www/svn/
	/archive			// 기존 파일 백업 전용
	/practice			// 연습 전용 
	/quintet			// QUINTET 프로젝트 관련 파일 전용

   
   A. archive 폴더는 기존 파일(VSS)을 저장하는 용도로 사용한다.
   B. pratice 폴더는 SVN 연습용으로 사용한다.
   C. quintet 폴더는 QUINTET 프로젝트 파일을 저장하는 용도로 사용한다.
   	C1. quintet 폴더의 하위에는 각 유닛별로 폴더를 생성한다. [단, 중복되는 유닛이 발생함으로 폴더의 이름은 "P/N(하위 3자리).UNITNAME" (예 - 130.MSG / 230.MON)으로 구성한다.]
	     (각 유닛의 경로가 너무 길어지지 않게 하기 위해서 장비별로 묶지 않고 나열한다.)
	C2. 각 유닛의 폴더안에는 목적별로 폴더를 구성한다.
	     [HW관련 소스코드는 "PLD"를, SW관련 소스코드는 "SW"를 Prefix를 사용한다. (만약 PLD가 여러개 이면, PLD1, PLD2와 같이 숫자로 구분한다.)]
 	     [유닛의 실제 동작을 위한 소스코드의 경우 "rel"을, 테스트를 위한 것이면 "test"를, 또 다른 목적의 코드가 있다면 적절한 단어를 Suffix로 사용한다.]
	     [Prefix 와 Suffix 사이는 "." 으로 구분한다. (예 SW의 실제 동작을 위한 소스코드 - "SW.rel" / HW의 첫번째 PLD의 테스트용 코드는 "PLD1.test" )]
	C3. 해당 폴더에 소스코드 및 파일을 업로드 하고 관리한다.
   D. binary 및 IDE의 workspace, solution, project 파일은 업로드 하지 않는다. 단, 만약을 대비하여 SW는 hex파일, HW는 pof파일은 업로드 한다.
